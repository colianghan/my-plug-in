前端菜鸟眼里的设计

用一句话形容前端和UI设计 ——

"在前端眼里，UI设计稿是动态的!"

我觉得，UI是用PS/AI等这些设计工具展示效果，而前端，则是用浏览器展示。

图层时代

我们眼里的世界 ——

  • 在公交地铁站边看到的巨幅广告图片
  • PM 眼中第一感觉是：这个设计不错！这个效果不错！为什么要这样设计呢？会吸引用户吗？
  • UI 眼中第一感觉是：这需要多少图层能完成？那个图片处理需要蒙板！这字体很不错！
  • 前端 眼中第一感觉是：如果把这个图片重构为 HTML 页面布局该如何？标签怎样嵌套会优化？CSS 兼容性又是如何？动态效果该如何？

前端与产品经理之间的交流 ——

  • PM：这个滑动效果能实现吗?
  • 前端：能
  • PM：这个按钮点击的交互呢?
  • 前端：能
  • PM：那这个背景色有圆角有阴影而且半透明，Hover之后有旋转效果的层呢?
  • 前端：额…

解释一下前端回答的两个能与一个额 ——

  • PM：这个滑动效果能实现吗? 这个按钮点击的交互呢?
  • 前端补脑 * 涉及到样式(css)和交互(js) * 页面布局能通过的浏览器：IE8 + Firefox3.5 + Chrome 9 + 等等 * IE7需要写个 hack(PS：幸好不用兼容IE6) * 或许要新写响应式 CSS 来兼容移动设备 * (能)
  • PM：那这个背景色有圆角有阴影而且半透明，Hover之后有旋转效果的层呢?
  • 额…(补脑中：如果是用 HTML5 与 CSS3 实现，so easy，但是 IE7，其实不建议做这么炫的效果。)
  • 前端：建议在主流浏览器上将效果展现到最佳，而低版本浏览器上不影响客户使用(优雅降级思想)。

前端与UI之间的关系 ——

以前端的角度看设计，或是以设计的角度看前端，公婆之争罢了，事事都是在矛盾中前行，阴阳只是互补，没有矛盾，没有阴阳，亦不会进步。

前端与UI如何更好的协作 ——

沟 通

总结一下子 ——

  • 1. 网站的页面是动态的。photoshop呈现的是静态的东西，而网站页面是动态的展现内容、布局和交互。设计师过多关注用户体验层面，很难对所有的细节做到面面俱到。而前端（包括开发）需要照顾到所有的功能点涉及到的页面，因此在前期要考虑的尽量周全，别让别人帮我们收拾烂摊子。
  • 2. 关注新技术。网页设计缺少技术支持永远只是艺术。设计师必须经常关注新的技术和交互方式，这样才能在设计的时候提供多种解决方案，才能权衡利弊找到最优化的方案。
  • 3. 界面元素的标准化和统一。前端关注代码的可重用性，设计师关注新创意。因此在设计前期就要考虑哪些元素和交互方式既可以满足用户体验又能够被重复使用，以此来提高效率。

总结一下子(续) ——

  • 4. 团队合作很重要。设计师很容易沉浸在自己的小世界里不能自拔，这是我们经常犯的通病。“沟通”是团队合作的关键，一切皆在沟通。
  • 5. 相信自己。前端通常出于不同的原因对一些交互方式可行性做出判断，比如代码复杂程度，技术可实现性等等。好的设计师需要有一些超前意识和冒险精神，当他们受 新技术的激发，认为它能够大大提升用户体验的时候，就需要把它当作挑战来实现。在对技术的深入了解后去说服前端一起努力实现。
